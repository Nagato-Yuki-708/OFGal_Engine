# 节点与执行流设计


以下内容仅供参考


## 节点
### 加减乘除
1. 名称：Add/Sub/Mul/Div
2. 输入引脚
- 执行入口
- 操作数a
- 操作数b
3. 输出引脚
- 执行出口
- 结果
```
需要对操作数类型准备函数的重载，比如乘可以让数字相乘（结果是数字），也可以string乘数字n（结果就是n个string连起来的字符串）
```
### if
1. 名称：if
2. 输入引脚
- 执行入口
- 布尔值
3. 输出引脚
- 执行出口a
- 执行出口b
```
如果布尔值为真，就执行a，否则执行b
```
### while循环
考虑工作量，我们丢弃for循环，只使用改进后的while循环
1. 名称：while
2. 输入引脚
- 执行入口
- 布尔值
3. 输出引脚
- 执行出口（连接到退出循环后要执行的第一个节点的执行入口）
- 循环体执行出口（连接到循环体的第一个节点的执行入口）
- 次数（执行次数，从0开始算，第一次循环中该数字为0）
```
每次执行循环体前检查布尔值，如果为false就退出循环，否则继续循环；
当循环体中break被执行就退出循环；
当循环体中continue被执行就立即进行下一次循环
```
#### break
1. 名称：break
2. 输入引脚
- 执行入口
3. 无输出引脚
#### continue
1. 名称：continue
2. 输入引脚
- 执行入口
3. 无输出引脚

### 入口节点
#### 开始运行BeginPlay
1. 名称：ShowtheFrame
2. 无输入引脚
3. 输出引脚
- 执行出口
#### 每N毫秒运行Play_per_N_ms
1. 名称：Play_per_N_ms
2. 输入引脚
- 毫秒（int，必须通过读取字面值来设置该值，否则统一视为1000）
3. 输出引脚
- 执行出口
#### 当某键按下时运行Play_when_N_push_down
1. 名称：Play_when_N_push_down
2. 输入引脚
- 按键类型（string，需要准备一个string,KeyCode枚举的映射表，KeyCode枚举定义在了KeyCode.h，必须通过读取字面值来设置该值否则视为无）
3. 输出引脚
- 执行出口
#### 当区域触发Play_when_triggered
如果该对象有触发区域组件的话，鼠标左键单击该区域时执行该节点，主要是判断鼠标坐标是否在区域中即可，区域大小不受缩放影响、坐标也是绝对坐标而不受父子坐标系影响
1. 名称：Play_when_triggered
2. 无输入引脚
3. 输出引脚
- 执行出口

## 执行流
蓝图系统被传入LevelData结构体和对应BlueprintData结构体，返回执行流
我们可以这么设计节点：
```cpp
class NODE{
public:
	NODE* lastNode;
	NODE* nextNode;
	vector<Pin_Data> InData;	//输入数据缓冲区
	vector<Pin_Data*> OutData;		//指向目标节点的数据缓冲区的指针
	
	T function(T& a,T& b,......);	//重载
	
	NODE* Loop;		//如果该节点不位于循环体中就设置为nullptr
};
```
还可以这么设计引脚
```cpp
class Pin_Data{
public:
	std::string oriType;	//决定了该引脚的原数据类型

	int V_INT;
	float V_FLOAT;
	bool V_BOOL;
	std::string V_STRING;
	Frame V_FRAME;
	//...
}
```
为什么要每个类型的变量都创建一遍？因为有时会涉及数据类型转换，所以当Pin_Data的某个值被设置后，其它类型的值都要做出对应处理（比如oriType为“int”，当V_INT从0设置为1234时，V_BOOL也要自动从false设置为true，...，其它类型的变量同理）


最后所有节点都被连接完毕，向游戏虚拟机传递入口节点实例的引用，就是传递了执行流的开头
